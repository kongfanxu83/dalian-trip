# 大连旅行规划 App — 项目文档

## 项目概述

面向大连旅行的一站式行程规划移动端 Web 应用。用户可以浏览大连 100 家精选商家（美食/景点/酒店/购物/娱乐），将感兴趣的地点加入行程，在地图上查看路线，并生成可分享的完整行程单。

---

## 技术栈

| 层级 | 技术 |
|------|------|
| 框架 | React 18 + Vite 5 |
| 路由 | React Router v6（HashRouter） |
| 样式 | Tailwind CSS v3 |
| UI 组件 | Radix UI + Lucide Icons |
| 拖拽 | @dnd-kit/core + @dnd-kit/sortable |
| 地图 | 高德地图 JS API v1.4.15 |
| AI 建议 | DeepSeek API（出行方式推荐） |
| 数据 | 本地静态数据（src/data/merchants.js，100条） |
| 状态持久化 | localStorage |
| 构建输出 | dist/ |

---

## 页面结构

```
/ (榜单页 Index)
  ├── 分类筛选：美食 / 景点 / 酒店 / 购物 / 娱乐
  ├── 商家列表（按 ranking_score 排序，TOP3 金银铜标）
  └── 购物车栏 → 跳转行程规划

/map (地图页)
  ├── 高德地图（全部100家商家标记，按区域坐标定位）
  ├── 分类过滤 Tabs
  ├── 已规划行程路线连线（按天分色虚线）
  ├── 点击标记 → 商家详情 + 加入行程（选择第几天）
  ├── 天数选择器（从今天起算）
  └── 底部横向推荐商家列表

/trip-planner (行程规划页)
  ├── 天数选择（1-30天，从今日起算）
  ├── 每天商家列表（可拖拽排序、跨天移动）
  ├── 商家间出行方式选择（驾车/地铁/步行/骑行 + AI推荐）
  ├── 每天预估费用
  ├── 保存行程 / 分享至发现大厅
  └── 底部推荐商家

/discovery-hall (发现大厅)
  └── 展示其他用户分享的行程

/profile (我的)
  └── 已保存的行程列表、收藏
```

## 核心数据流

```
用户浏览商家
    ↓
加入购物车（useCart hook → localStorage.cartItems）
    ↓
点击"去规划" → /trip-planner
    ↓
选择天数 → 生成日期 key（格式：月-日）
    ↓
dateItems 状态（按天分组商家）→ localStorage.tripDateItems
    ↓
地图页读取 tripDateItems → 画路线连线
    ↓
保存行程 → localStorage.myTrips
分享行程 → localStorage.sharedTrips
```

## 商家数据说明

- 文件：`src/data/merchants.js`
- 来源：CSV 导入（merchant_rankings_2026-03-14.csv）
- 数量：100 家
- 字段：id, name, category, rating, price, area, tag, address, description, ranking_score, is_featured 等
- 坐标：无真实经纬度，地图页用 `area` 字段映射大连各区中心坐标 + 确定性偏移生成

## 部署

项目为纯静态 SPA，构建后上传任意静态托管即可：

- **Vercel**（推荐）：连接 GitHub 仓库，自动识别 Vite，零配置部署
- **Netlify**：Build command `npm run build`，Publish directory `dist`
- **任意 CDN / 对象存储**：上传 `dist/` 目录内容

路由使用 HashRouter（`/#/path`），无需服务器配置重定向规则。
